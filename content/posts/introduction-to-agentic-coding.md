## 核心问题：从碎片化建议到编排瓶颈

AI 辅助编程的每次范式迭代都试图降低认知负荷，但早期版本引入了显著的**手动编排税**。

### 上下文范围鸿沟

- **传统 AI 自动补全**：主要在文件级操作。能生成 REST 样板代码，但缺乏**全局变量和数据流可见性**——无法追踪底层工具服务的变更如何在分布式系统中传播
- **对话式 AI**：提供更深的架构分析，但需要手动注入上下文——开发者必须策划代码片段、日志和 schema 来"锚定"模型，高摩擦且推理碎片化

### 手动编排税

痛点不在生成代码的质量，而在系统编排的负担：手动在数十个依赖文件中传播架构变更、识别每个受影响的调用点、实现-编译-捕获错误-重新粘贴的高延迟反馈循环。

## Agentic 工作流：四阶段操作循环

### 阶段 1：上下文收集与映射

Agent 建立本地环境的全面地图：分析 `package.json`/`requirements.txt` 等项目清单，理解技术栈，并**模仿本地项目约定**——确保新实现不仅语法正确，且与现有代码库的风格和结构模式习惯一致。

### 阶段 2：自适应规划

不执行静态预定义指令，而是构建动态实现策略。如需修改 GraphQL resolver，系统识别 schema、数据访问层和认证中间件之间的关系。计划是非线性的，根据发现阶段的环境反馈动态调整。

### 阶段 3：实现循环

以终端为"手"，在文件系统中执行幂等修改。不同于 IDE 扩展在单个缓冲区内建议修改，Agentic 系统执行**多文件协调**——同时更新 handler、重构数据库 schema、调整 API 客户端签名，确保项目级一致性。

### 阶段 4：验证

Agent 直接与本地工具链交互——运行 `npm test`、`pytest` 或自定义构建脚本。如果返回堆栈跟踪或断言失败，Agent 启动迭代"写-测-调"循环，独立精炼代码直到环境状态匹配初始需求。

## Claude Code 的设计决策

### 终端优先执行

作为 CLI 工具部署（`npm install -g @anthropic-ai/claude-code`），绕过浏览器和 IDE 的沙盒限制。直接在 shell 中操作，无需人类中介即可与本地编译器、OS 级工具和项目构建环境交互。

### MCP（Model Context Protocol）

作为**可扩展上下文提供者**，将模型与特定工具链解耦。Claude Code 可从外部 issue tracker、文档站点和内部系统摄取数据，提供统一的上下文层。

### 许可与控制模型

平衡高自主性和工程安全：Agent 在任何文件修改前呈现结构化 diff 和摘要执行计划。执行自主，但最终架构授权在人类。

### CLAUDE.md 状态持久化

作为架构约束的**状态持久层**——记录编码标准、命名约定和项目特定技术债策略，确保 Agent 跨会话保持上下文一致。

## 三代 AI 编程对比

| 维度 | AI 自动补全 | 对话式 AI | Agentic Coding |
|------|-----------|----------|---------------|
| 上下文范围 | 文件/片段级 | 粘贴的代码段 | **项目级（整个代码库）** |
| 实现自主性 | 被动建议 | 迭代指导 | **自主执行** |
| 工具集成 | IDE 限制 | 无（基于 Web） | **本地环境（CLI、Git、编译器、测试、MCP）** |
| 核心价值 | 代码段生成速度 | 架构分析 | **持续的自主任务完成** |

延迟洞察：传统自动补全提供亚秒反馈，Agentic Coding 因多轮推理和迭代工具使用有显著计算开销。但每步增加的"完成时间"大幅减少了复杂特性的**总人力投入**。

## 实证：乐天案例

在 vLLM 开源库中实现激活向量提取方法：

- **系统复杂度**：1250 万行代码，Python/C++/CUDA 多语言栈
- **持续自主工作**：7 小时会话，工程师仅提供偶尔的高层指导
- **数值精度**：最终实现与参考方法达到 **99.9% 数值精度**
- **吞吐提升**：特性交付加速 **79%**，24 天开发周期缩至 5 天，支持 5x 并行任务执行
