Continue 这篇 [Introducing Instinct](https://blog.continue.dev/instinct/) 在我看来很重要，不只是因为它发布了一个新的 Next Edit 模型，而是因为它把这件事从“少数产品团队内部可做的魔法功能”往前推了一步，变成了一个开源社区也可以参与的工程方向。

如果 Zed 那篇文章更像是在证明“编辑预测是个成立的新任务”，那 Instinct 这篇更像是在回答另一个问题：**既然这个任务成立了，开源社区能不能用真实编辑数据、合理的训练方法和公开评估，把它做成一个持续演进的开放路线？**

## Instinct 真正试图证明的，不是模型多强，而是这条路线能开源

文章里一开始就强调，Next Edit 这件事和传统 autocomplete 不是一回事。自动补全只能在光标处插入文本，而 Next Edit 要处理的是删除、替换、结构重组和多步重构。这里最关键的不是“生成得更多”，而是“是否理解了开发者接下来要改什么”。

Continue 想证明的，其实不只是 Instinct 在 benchmark 上超过了 Zeta，而是这类能力不必永远封闭在产品内部。你可以收集真实编辑轨迹、公开训练数据、开放模型权重，再围绕接受/拒绝反馈继续优化它。

这件事的重要性在于，它把 next-edit 从一个“编辑器厂商专属 feature”变成了一个有机会形成公开数据集、公开方法和公开比较基线的方向。对整个生态来说，这比单次分数更有价值。

## 它最有分量的部分，是训练数据而不是模型本身

我觉得这篇文章最值得反复看的部分，其实是数据工程。因为对于 Next Edit 这种任务来说，最难的从来不是“挑一个好底模”，而是“到底什么叫一条合格的编辑样本”。

Continue 的选择很明确：不用 git commit 这种过于粗糙的快照差异，而是收集真实开发会话，把最近几次编辑轨迹、当前文件、外部上下文以及可编辑区域一起喂给模型。这个思路很重要，因为 Next Edit 的核心不是代码本身，而是代码正在如何变化。

文章里还提到，他们对“编辑”这个单位做了很多启发式定义，比如如何把零散按键聚合成自包含 diff，如何过滤掉来回来去的乒乓式无效修改。这其实说明了一件事：真正的训练难点不在模型结构，而在于你能不能把人类编辑行为转写成对模型有意义的序列。

从这个角度看，Instinct 更像是一项数据建模工程，而不是单纯的模型发布。

## SeleKT 的价值，在于它试图少忘一点

训练方法上，Continue 没走最常见的 LoRA 路线，而是采用了 SeleKT。我觉得这里的关键不只是“更新 5% 参数”这种数字，而是它背后的意图：不要预设哪些参数该被改，而是先让训练过程自己发现，Next Edit 这个任务真正需要改动哪些地方。

这个选择很有意思，因为它直击一个开源模型训练里很现实的问题。你当然可以通过强力微调让模型更贴合新任务，但与此同时，你也可能把它原本的通用编程能力磨损掉。SeleKT 的吸引力就在于，它试图在“学会 next-edit”与“别忘掉已有 coding knowledge”之间找一个更平衡的位置。

我觉得这也是 Continue 和很多产品团队思路不同的地方。它不只是想把功能做出来，还在认真处理“如何把一个基础模型适配成编辑模型，同时尽量不伤到底座能力”这个训练问题。

## 评估设计说明了 open benchmark 应该怎么做

Continue 继续沿用了 LLM Judge 的路线，但它在评分定义上更有意思。尤其是给 3 分留出了一个空间：这不是 ground truth，但它可能依然是一个经验丰富的开发者会做出的合理编辑。

我非常认同这个设计。因为对 Next Edit 来说，真正重要的不只是“是否复现了开发者当时的准确操作”，还包括“这是不是一个符合当前上下文的专业级编辑”。如果没有这个中间层，评估就会过度奖励完全匹配，而低估很多合理替代方案。

这也是为什么我觉得这篇文章对开源生态很有价值。它不仅开了模型和数据，还在尝试把评估口径讲得更清楚。对于一个新任务来说，公开评估标准本身就是基础设施。

## 真正让我在意的，是它把本地部署和开放迭代绑在了一起

文章里还有一条很现实的线索：Instinct 是可以本地跑的。表面看这是隐私和自部署需求，实际上它背后代表的是另一种开发者权力结构。

如果 Next Edit 只能跑在封闭云端服务里，那开发者只能被动使用厂商给的能力；但一旦它可以本地部署、微调、叠加自有 accept/reject 数据再优化，那团队就有机会把“编辑预测”变成自己的生产力资产，而不是只依赖某个供应商的黑箱。

这也是为什么文章最后提到 KTO 等继续优化路线时，我会觉得比“开源了一个 7B 模型”更有意义。真正值得关注的，不只是你今天拿到了一个模型，而是你有没有拿到一条可以继续演化的链路。

## 我的看法：Instinct 的意义，在于它让 Next Edit 从 feature 变成 open problem

我读完之后最大的感受是，Instinct 这篇文章真正重要的地方，不在于它今天比某个模型多拿了几点，而在于它把 Next Edit 这件事从“产品能力展示”推进成了一个公开可讨论、可复现、可持续优化的开放问题。

这对开源生态很关键。因为如果所有 next-edit 能力都只存在于闭源产品里，社区很难形成稳定的比较基线、训练共识和迭代方法。而 Instinct 至少让大家看到，这件事不只能被做出来，还能被开源地做出来。

所以如果要用一句话总结这篇文章，我会说：Instinct 的真正价值，不是又多了一个开源代码模型，而是开源社区终于开始有机会围绕“开发者编辑轨迹”这件事，建立属于自己的训练和评估体系。
